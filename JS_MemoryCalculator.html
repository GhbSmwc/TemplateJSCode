Note: This javascript calculator uses BigInt as its number format. Make sure your browser supports this format. Currently (the time of writing 2019-12-29), edge, IE (why use this browser?), safari (and IOS) don't support this format. The reason is because to prevent a potential rounding issues (more specifically, precision loss), especially with division (even with the Math.floor(x/y) function, it can round the number upwards before the floor function rounds it downwards). An example is this: Math.floor(100000000000000.9999), its between 100000000000000 and 100000000000001, however, it is rounded towards 100000000000001 and not 100000000000000, which is wrong.
